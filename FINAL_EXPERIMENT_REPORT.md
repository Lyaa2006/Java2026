# 实验报告：作业互助批改平台（网页版）

## 一、项目基本信息

- 项目名称：作业互助批改平台
- 最终版本形态：Web 版
- 补充说明：项目开发过程中曾实现过 Java Swing 桌面版，当前仓库中仍保留该版本代码，以便后续桌面端开发需要；但本次实验报告所描述和提交的最终成果，以当前网页版本为准。

## 二、选题依据

### 1. 选题背景

在课程作业批改场景中，教师和学生通常会遇到几个实际问题：
1. 传统通过聊天软件或邮箱收发作业时，文件版本容易混乱；
2. 教师批改后的反馈往往分散在文件、文字消息和口头说明中，学生难以集中查看，形成体系；
3. 传统的作业提交平台，教师批改和学生订正只能通过评语或者下载原文件修改后上传进行，对于需要小范围逐行批改订正的作业不够友好。
4. 部分传统作业提交平台只支持单个文件上传，如学生需要提交多份文件则需要以压缩包的形式上传

基于以上问题，本项目选择设计并实现一个轻量级的“作业互助批改平台”。平台围绕“按班级管理、教师布置作业、学生提交作业、教师批改反馈、学生在线订正”这一核心流程展开，既贴近教学实际，具有较强的实际意义。

### 2. 选题目标

本项目希望实现以下目标：

- 支持学生与教师双角色登录与注册。
- 支持学生加入班级提交作业（一次作业可上传多个文件），不同班级作业访问权限严格隔离，避免不同班级之间的作业混淆。
- 支持教师创建班级，发布作业、管理班级的作业、上传批改文件并填写评语（支持分文件评语和总评）。
- 支持文本类作业进行在线批改与在线订正。
- 支持对原始内容、教师批改内容、学生订正内容进行颜色区分。

### 3. 与类似竞品之间的区别

本项目并不是单纯模仿现有教学平台，而是针对课程实验场景做了轻量化与功能聚焦的设计。与类似产品相比，主要区别如下。

| 对比对象 | 常见特点 | 本项目的区别 |
| --- | --- | --- |
| 学习通、雨课堂等综合教学平台 | 功能全面，覆盖课程通知、签到、考试、作业等多个模块，但系统较重，部署与使用依赖平台生态 | 本项目只聚焦“作业提交-批改-订正”主流程，功能更集中，更适合课程实验和本地演示 |
| 邮箱、QQ、微信等文件收发方式 | 使用门槛低，但文件版本容易混乱，反馈分散，缺少结构化管理，并且依赖文件传输 | 本项目将提交记录、作业状态、教师评语、批改文件、在线订正整合在一个界面内，并且支持多文件上传、在线批改 |
| 批改网、在线 OJ 等自动评测平台 | 更偏向标准答案比对、自动评分或程序评测 | 本项目聚焦教师人工批改、学生二次订正，以及文本类作业的过程性反馈，不依赖自动判题 |

进一步来看，本项目的差异化体现在以下几个方面：

- 在线批改机制：既支持教师上传批改后的文件，也支持文本类作业的在线批改与在线订正。
- 过程可视化：通过黑色、红色、蓝色三色区分，直观反映“原文-批改-订正”的演进过程。
- 多文件上传：每位学生在提交作业时可以上传多个文件，无需打成压缩包，并可在提交后对单个文件实施更新、删除；教师可以在线分别查看、批改、下载提交下的单个文件
- 班级权限控制：教师可以分班级管理学生作业。


## 三、概要设计说明

### 1. 总体设计思路

本项目最终采用 B/S 架构的轻量级实现方式，整体由前端交互层和数据存储层组成。当前版本以浏览器端数据管理为主，用户、班级、作业和文件等信息通过 IndexedDB 进行组织与持久化，从而保证系统主流程可以独立完成；同时，这种分层方式也为后续接入远端数据库和后端服务保留了扩展空间。

系统总体可以分为三层：

- 前端：负责登录、注册、学生端、教师端、弹窗、表格、在线编辑区等界面展示。
- 业务层：负责用户认证、班级申请与审批、作业提交、权限校验、在线批改、在线订正等逻辑处理。
- 数据层：负责当前版本的数据结构设计、对象仓库创建、事务读写、文件 Blob 保存和业务数据持久化，并可继续扩展为远端数据访问层。

### 2. 模块结构设计

当前 Web 版本主要由以下文件构成：

| 模块 | 主要文件 | 作用 |
| --- | --- | --- |
| 页面入口 | `index.html` | 负责页面骨架、资源引入和应用挂载 |
| 前端交互层 | `web/app.js` | 负责登录注册、学生端、教师端、弹窗、在线编辑、锁机制和页面渲染 |
| 数据存储层 | `web/storage.js` | 负责当前数据访问逻辑封装、业务数据读写、文件存储、权限判断和核心业务函数 |
| 样式层 | `web/styles.css` | 负责整体页面布局、表单、表格、弹窗、编辑区域、颜色标识等视觉效果 |

### 3. 数据设计说明

当前版本在数据层中建立了多个数据仓库，用于存储不同类型的数据：

| 数据仓库 | 作用 |
| --- | --- |
| `users` | 保存用户账号、密码、姓名和角色信息 |
| `submissions` | 保存作业提交记录、状态、班级信息、评语、在线编辑内容等 |
| `files` | 保存原始作业、批改文件、订正文件等二进制或文本文件 |
| `classes` | 保存班级信息、创建者、授权教师、成员列表、待审批学生列表 |

其中，作业记录是系统的核心数据对象，主要包含以下信息：

- 学生账号与姓名
- 班级编号与班级名称
- 提交内文件列表（原始作业文件名+文件索引）
- 批改文件列表（批改文件名+文件索引）
- 订正文件列表（订正文件名+文件索引）
- 作业状态
- 教师评语

### 4. 业务流程设计

系统的主要业务流程如下：

1. 用户进入系统后，可根据身份选择登录或注册。
2. 教师可创建班级，并可为其他教师授予班级管理权限。
3. 学生需先申请加入班级，待教师审批通过后才能向该班级提交作业。
4. 教师选择对应班级布置作业，该班级学生可上传一个或多个文件，系统生成提交记录并保存原始文件（文本字体显示为黑色）。
5. 教师查看有权限的班级作业后，可选择上传批改文件，或对文本类作业进行在线批改。（批改字体显示为红色）
6. 学生可下载批改文件，也可对文本类作业进行在线订正。（订正字体显示为蓝色）

## 四、已实现的主要功能

系统已经实现了以下主要功能。

### 1. 用户与身份功能

- 支持学生、教师双角色注册。
- 支持按角色进行登录校验，避免学生账号误登教师入口。
- 支持会话级登录状态保持，刷新页面后可恢复当前登录用户。

### 2. 班级管理与权限控制

- 教师可以创建班级，并填写班级说明。
- 学生可以查看可加入班级并提交加入申请。
- 教师可以对学生的入班申请进行同意或拒绝处理，同意加入后学生方可提交作业。
- 班级创建者可以授权其他教师共同管理班级。
- 教师只能查看自己有权限的班级作业，防止越权访问。

### 3. 作业提交与多文件处理

- 教师在自己管理的班级布置作业后，该班级学生可提交作业。
- 学生可上传**一个或多个**文件，进行提交。
- 教师可以对于学生上传的文件进行在线批改，填写评语；也可以对作业整体进行评语或上传批改后的文件。
- 学生可以下载教师上传的批改文件。

### 4. 在线批改与在线订正

- 对 `txt`、`java`、`md`、`csv` 等文本类文件提供**在线查看与编辑能力**。
- 教师可以直接在页面中对文本内容进行在线批改（字体显示为红色）。
- 学生可以基于教师批改结果继续在线订正（字体显示为蓝色）。
- 学生不能修改教师的红色批改内容，教师也不能修改学生的蓝色订正内容。
- 保存后系统会自动生成对应的批改文件或订正文件记录，保证作业状态可回溯。
ps：系统引入行锁机制，两个用户不可以同时改同一行文件。

## 六、所使用的关键技术

### 1. 前端界面与交互设计

当前 Web 版实现了登录注册、学生端、教师端、作业列表、弹窗和在线编辑区等界面，保证了系统具备完整的可视化交互能力。用户可以直接在页面中完成注册、登录、建班、提交、批改和订正等主要操作。

底层核心知识：图形界面设计 GUI

### 2. 数据持久化与数据组织

系统当前使用 IndexedDB 保存用户、班级、作业、文件和权限等信息，并按对象仓库进行分类管理。这样既保证了数据读写的完整性，也让整体数据结构更清晰，便于后续继续扩展。

底层核心知识：Java 集合框架、并发编程和 MVC 模式

### 3. 文件处理与输入输出

项目支持学生上传作业、教师上传批改文件、学生下载结果，并能将在线编辑后的文本重新生成为文件。对文本文件和普通文件分别进行读取、保存和回写，是平台形成完整作业流转的基础。

底层核心知识：二进制输入输出、网络编程

### 4. 在线批改、文本分行与内容合并

对于 `txt`、`java`、`md`、`csv` 等文本类作业，系统支持在线批改和在线订正，并把原始内容、教师批改内容、学生订正内容统一组织后再显示为不同颜色。为了实现这一点，系统需要对文本进行分行、比对、合并和重组。

底层核心知识：对象和类

### 5. 权限控制、角色隔离与异常处理

系统围绕教师和学生两个角色设计了不同权限。学生需要先加入班级，教师只能查看有权限的班级作业，同时系统会对空文件、重复申请、越权访问和非法参数等情况进行校验并提示。

底层核心知识：继承封装和多态、抽象类和接口、泛型和异常处理

### 6. 协作锁机制与并发控制

在在线编辑部分，系统通过锁信息同步来限制同一行内容的重复编辑，避免多人或多标签页同时修改同一位置而造成冲突。锁的归属、状态刷新和超时释放共同保证了编辑过程的基本有序性。

底层核心知识：多线程、基于多线程的网络编程和并发管理

### 7. 分层设计、模块解耦与可扩展架构

当前项目已经形成了较清晰的分层结构：页面负责展示与交互，业务逻辑负责处理规则，数据层负责存储和查询。这种结构让系统职责清楚，也为后续接入后端服务和远端数据库保留了良好的扩展空间。

底层核心知识： Web 应用相关内容（SpringBoot）、并发编程和 MVC 模式

## 七、程序亮点

结合实验目标与最终实现，本项目的主要亮点如下：


### 1. 兼顾批改文件上传下载与文本在线批改

项目既支持上传和下载批改文件，也支持文本类文件的在线查看、在线批改和在线订正，比单纯的作业托管平台更接近实际教学需求，为简单文本作业（如Java课程的小作业提供了便利）。

### 2. 三色内容来源区分

系统通过不同颜色区分原始内容、教师批改内容和学生订正内容，使批改过程更加直观。教师能快速看到学生改动，学生也能清晰分辨教师意见和自己的修改位置。

### 3. 多文件提交管理机制

系统支持同一作业提交多个文件，学生可以对于单个文件进行更新或者删除；教师可以对于提交中的单个文件进行在线批改、填写评语；也可以对于整体提交进行评语。避免了传统多文件提交需要使用压缩包的问题，实现更为便捷和细粒度的作业管理。

### 4. 班级维度权限设计较完整

学生先加入班级才能查看并提交作业，教师只能访问有权限班级的作业，班级创建者还可以授权其他教师共同管理，适配多教师、多课程的真实教学管理场景，具有真实落地的潜力。
