## 一、选题名称

**题目**：**基于Java Socket的作业互助批改平台**

**一句话定位**：一个C/S架构的小型教学辅助系统，支持学生上传作业、教师在线批注与回传，并通过多线程与文件传输实现实时反馈。

**选题理由**（覆盖所有作业要求）：

- 体现Java特点（面向对象、集合、异常、IO）
- 体现高级部分（NIO、并发、反射、自定义注解）
- 多线程（服务端并发处理、异步通知）
- 网络（TCP自定义协议）
- 上传下载文件（作业 + 批注文件）
- 设计模式（工厂、模板方法、状态、观察者）

------

## 二、核心功能

| 角色 | 功能                        | 技术点                         |
| :--- | :-------------------------- | :----------------------------- |
| 学生 | 注册/登录                   | 账号文件存储                   |
| 学生 | 上传作业（任意文件）        | 分块上传 + 校验                |
| 学生 | 查看已批改作业              | 文件下载 + 本地打开 /在线查看           |
| 教师 | 查看所有提交的作业          | 文件列表刷新（多线程扫描）     |
| 教师 | 在线批改作业          | 文件列表刷新（多线程扫描）     |
| 教师 | 下载学生作业                | 断点续传（可选）               |
| 教师 | 上传批注文件（如PDF批注版） | 与原始作业关联                 |
| 系统 | 作业状态管理                | 状态模式：已提交/批改中/已批改 |
| 系统 | 批改完成通知                | 观察者模式 + 异步消息          |

## 三、关键技术点与设计模式

### 1. 网络通信（TCP + 自定义协议）

- 每条消息格式：`类型|长度|内容`
- 类型包括：登录、上传请求、文件块、批注上传、心跳

### 2. 多线程设计

- **服务端**：主线程接受连接，每条连接交给 `线程池` 处理
- **通知机制**：使用 `BlockingQueue` 存放待通知的学生列表
- **UI不阻塞**：客户端上传在后台线程执行

### 3. 设计模式具体落地

| 设计模式   | 应用场景                                                     |
| :--------- | :----------------------------------------------------------- |
| 工厂模式   | 根据消息类型创建不同的请求处理器（如 `LoginHandler`、`UploadHandler`） |
| 模板方法   | 定义文件处理的通用流程（校验→接收→存储→通知），子类实现差异  |
| 状态模式   | 作业状态流转，避免大量 if-else                               |
| 观察者模式 | 教师批改完成后，通知所有在线学生端                           |
| 单例模式   | 在线用户管理器、配置管理器                                   |

### 4. Java高级特性

- **NIO**（可选简化版）：使用 `FileChannel` 高效读写文件
- **自定义注解**：标记哪些方法需要登录验证
- **并发集合**：`ConcurrentHashMap` 存储在线用户
- **MVC**: 教师端和学生端视图同时查看修改

## 四、可能亮点

| 亮点             | 说明                                   |
| :--------------- | :------------------------------------- |
| **断点续传**     | 上传中断后可从断点继续                 |
| **批注自动关联** | 教师上传的批注文件自动与学生作业对应   |
| **离线通知缓存** | 学生下次登录时自动收到批改完成通知     |
| **心跳检测**     | 自动检测客户端异常断开，释放资源       |
| **配置文件化**   | 服务器端口、存储路径等可配置，不硬编码 |
| **简单日志系统** | 记录所有上传/下载操作，便于排错        |

**建议你重点讲“断点续传”或“离线通知缓存”**，这两个最能体现多线程和文件处理的深度。
